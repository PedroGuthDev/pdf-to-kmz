# Summary: Web Research — External Techniques for GPS Coordinate Accuracy

## Outcome

Research complete. Full findings in `20260519-RESEARCH.md`.

### Key findings

1. **HMM + Viterbi (Newson & Krumm 2009) is the correct algorithm** to replace N3's beam search. Beam search prunes correct branches when local score is poor; Viterbi never prunes. For n=14 posts, k=20 candidates: 5,600 operations, ~30 lines of plain JS, no library needed.

2. **N1 (cable arc-length walk) is confirmed as the primary fix.** Arc-length parameterization is the standard pattern (SVG `getPointAtLength`, `pointAtArcLength` already in `cable-builder.js`). Theoretical error budget < 2 m per page. Should be enabled as default (not opt-in) when Cabo Projetado is detected.

3. **PDF coordinate precision is NOT the problem.** 1 PDF point = 0.36 m → float precision floor < 0.00004 m. The 5–36 m errors are 5 orders of magnitude above this. The errors are algorithmic (mis-assignment).

4. **SIRGAS-2000 datum introduces no error.** The project already uses UTM from the printed PDF grid which is in SIRGAS-2000. No datum conversion needed.

5. **Similarity (Helmert) transform is correct for CAD PDFs.** The current 4-DOF (Tx, Ty, scale, rotation) is the right choice. The 26° shear in N5's affine fit is a diagnostic of bad input points, not real shear — confirming N1 must fix inputs before N5 can help.

### Priority order from research
