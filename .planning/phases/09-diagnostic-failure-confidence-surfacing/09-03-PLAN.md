Surface the already-computed confidence and failure signals in the browser UI (the final
wiring wave). Add a dedicated status banner (D-07) that renders `dwgConfidence.overall` as a
Portuguese tier label (SC-1), the hard-failure reason + nearest-region hint when

`postTiers` into `lastCalcResult` so `buildKml` can color by tier (Pitfall 6). No `%` anywhere
(CONF-04).

`result.hardBlock` and `dwgConfidence.overall`; Plan 02 produced `stats.unresolvedNoCoord` and
tier-aware `buildKml(..., { postTiers })`. Plan 03 reads those and renders them.
Output: a confidence banner DOM block + the main.js wiring that gates the KMZ and renders the
overall tier, hard-block reason, and unresolved list.
</objective>
