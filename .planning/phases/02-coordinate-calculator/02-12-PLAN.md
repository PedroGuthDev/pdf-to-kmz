    STEP 2 — Build a fake regionLibrary in Node using fake-indexeddb:
    ```javascript
    const idbFactory = new IDBFactory();
    const regionLibrary = createRegionLibrary(idbFactory);
    const dxfBlob = new Blob([readFileSync('./siriu.dxf')], { type: 'text/plain' });
    await regionLibrary.addRegion('siriu', dxfBlob);
    console.log('[dwg-harness] Region "siriu" added to library.');
    ```

    NOTE: The Blob constructor requires a Uint8Array or ArrayBuffer in Node. Use: new Blob([readFileSync('./siriu.dxf').buffer]) or check that the Buffer produced by readFileSync is accepted by Blob. If Blob is not available globally in Node (< 18.x), add: import { Blob } from 'node:buffer'; at the top.

    STEP 3 — Build synthetic PDF topology from ground truth:
    Fabricate posts, distances, and connections arrays that mimic a PDF's output for the siriu route:
    - fakePosts: gt.map((g, i) => { const { easting, northing } = latLonToUtm(g.lat, g.lon); return { number: g.number, x: easting - refE, y: northing - refN, pageNum: 1 }; }) where refE = latLonToUtm(gt[0].lat, gt[0].lon).easting and refN = northing of post 01. This places posts in a coordinate system relative to post 01. This is a rough approximation of PDF coordinates; the pairing algorithm uses these only to compute bearing/distance predictions.
    - fakeDistances: for each consecutive pair (i, i+1), compute meters = haversineMeters(gt[i].lat, gt[i].lon, gt[i+1].lat, gt[i+1].lon). Build: [{ from: gt[i].number, to: gt[i+1].number, meters }].
    - fakeConnections: sequential [{ from: gt[i].number, to: gt[i+1].number, gap: false, cross_page: false }] for i in 0..28.
    NOTE: The siriu ground truth has 85 posts but the route may have branches (posts 10, 11 are on a branch per the file layout showing posts 10-18 with different lat/lon pattern). Inspect the ground truth coordinates to determine the topology. If the coordinates suggest a branch (e.g. consecutive posts that jump far spatially), detect this and add branch connections. For simplicity, treat the first 30 posts as linear unless the harness fails — the acceptance gate will reveal topology issues.
