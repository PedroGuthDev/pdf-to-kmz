<objective>
Implement the two core DWG modules: the DXF entity loader and the strict pairing algorithm. These are the foundational building blocks that region-library.js (02-10) wraps and that tests (02-11) exercise directly.

Purpose: The DXF loader translates raw dxf-parser output into the domain objects (UTM post positions, adjacency-hint edges) that the pairing algorithm consumes. The pairing algorithm implements D-DWG-PAIR-02 — the anchor + PDF-topology walk that maps each PDF post to a DWG INSERT.

Output: parser/dwg/dxf-loader.js and parser/dwg/region-pairing.js — both pure functions, no IndexedDB dependency, testable in Node.

// Custom subclass to define the bbox extraction from a point:
class PostIndex extends RBush {
  toBBox(post) { return { minX: post.x, minY: post.y, maxX: post.x, maxY: post.y }; }
  compareMinX(a, b) { return a.x - b.x; }
  compareMinY(a, b) { return a.y - b.y; }
}
const tree = new PostIndex().load(posts); // bulk O(n log n) insert
// Nearest-in-bbox query:

const dump = tree.toJSON();

    Imports:
    - import RBush from 'rbush';

    Exports:
    - export const DEFAULT_TOLERANCE_M = 15;
    - export const GAP_TOLERANCE_M = 25;
    - export const ADJACENCY_SNAP_M = 3; (tolerance for building adjacency graph from cable endpoints)

    export function buildPostIndex(posts):
    - Define class PostIndex extends RBush with toBBox, compareMinX, compareMinY methods (per research §4 build pattern).
    - Return new PostIndex().load(posts) (bulk load). The loaded tree's items have the same shape as the input posts[] — each item is { x, y, block }.

    export function buildAdjacencyGraph(posts, cableEdges):
    - For each cable edge { a, b }, find the index in posts[] of the nearest INSERT to a.x/a.y (within ADJACENCY_SNAP_M) and to b.x/b.y (within ADJACENCY_SNAP_M). If both endpoints snap to distinct INSERTs (iA !== iB), record the pair: adjacency.get(iA).add(iB) and adjacency.get(iB).add(iA).
    - Return Map<number, Set<number>> keyed by index into posts[].
    - Use a brute-force O(n×m) scan (483 posts × 451 edges = 217,683 comparisons — negligible; no rbush needed here).

    export function pairPostsAgainstRegion({ posts, distances, connections, startLat, startLon, region, postIndex, adjacencyGraph, warnings }):
    - Implements D-DWG-PAIR-02 algorithm. See 02-RESEARCH.md §5 pseudocode for the full walk.
    - Step 1: latLonToUtm(startLat, startLon) → { easting: anchorE, northing: anchorN }. Cross-check that latLonToUtm returns zone === region.crs.zone; if not, push { kind: 'dwg-zone-mismatch', expected: region.crs.zone, got: zone } to warnings and return { ok: false, failedAt: 1, nearestDistance: null }.
    - Step 2: postIndex.search bbox ± DEFAULT_TOLERANCE_M around anchorE/anchorN. If empty, push D-DWG-FAIL-01 warning (kind: 'dwg-pair-fail', at_post: posts[0].number, predicted: { lat: startLat, lon: startLon }, nearest_dwg_distance_m: null, tolerance_m: DEFAULT_TOLERANCE_M) to warnings. Return { ok: false, failedAt: posts[0].number, nearestDistance: null }.
    - If anchor candidates exist, pick the single nearest (min Math.hypot distance to anchorE/anchorN). If distance > DEFAULT_TOLERANCE_M, fail with same warning structure.
    - Step 3: Build ordered walk sequence from connections array. Use the connections array (from, to, gap, cross_page fields) to determine the PDF-order walk. Sort the walk to follow logical sequence, respecting branch splits. For branches: when a post has multiple outgoing connections, pair both downstream sub-walks independently starting from the same dwg[from] anchor.
    - Step 4: For each edge (from→to) in the ordered walk:
      * Resolve PDF bearing from posts array (find posts[from].x/y and posts[to].x/y, compute atan2 then convert to compass degrees with flipY convention matching coordinate-calculator.js — in PDF space +y is downward/south, so bearing = atan2(dx, dy) NOT atan2(dy, dx)).
      * Resolve meters from distances array (find entry with from===edge.from && to===edge.to, use .meters field).
      * If meters is missing, skip this edge (cannot predict without a distance — emit warning kind: 'dwg-missing-distance').
      * Compute UTM displacement: dE = meters × sin(bearingRad), dN = meters × cos(bearingRad). NOTE: In zone 22S near -27.97°, grid convergence is ~0.5° — do NOT apply a convergence correction in v1.
      * predE = dwgByPostNumber.get(from).x + dE, predN = dwgByPostNumber.get(from).y + dN.
      * tolerance = edge.gap ? GAP_TOLERANCE_M : DEFAULT_TOLERANCE_M.
      * candidates = postIndex.search(predE ± tolerance, predN ± tolerance).
      * If empty: push dwg-pair-fail warning, return { ok: false, failedAt: to, nearestDistance: null }.
      * Score candidates: for each candidate c, compute rawDist = Math.hypot(c.x - predE, c.y - predN). Compute cIdx = region.posts.indexOf(c). isNeighbour = adjacencyGraph.get(region.posts.indexOf(dwgByPostNumber.get(from)))?.has(cIdx). score = isNeighbour ? rawDist * 0.5 : rawDist. Pick best = min score candidate.
      * If Math.hypot(best.x - predE, best.y - predN) > tolerance: push dwg-pair-fail warning with actual distance. Return { ok: false, failedAt: to, nearestDistance: dist }.
      * Collision check: if claimedInsertsSet.has(best): push { kind: 'dwg-pair-collision', at_post: to } to warnings. Return { ok: false, failedAt: to, nearestDistance: 0 }.
      * Accept: dwgByPostNumber.set(to, best), claimedInsertsSet.add(best).
    - Step 5: Convert all paired DWG INSERTs to lat/lon: for each post p in posts[], const { lat, lon } = utmToLatLon(dwgByPostNumber.get(p.number).x, dwgByPostNumber.get(p.number).y, region.crs.zone). Collect into coords array { postNumber: p.number, lat, lon, source: 'dwg', dwg_block: dwgByPostNumber.get(p.number).block }.
    - Return { ok: true, coords }.

    PDF bearing convention (CRITICAL — must match how pdf-parser.js emits post x,y positions):
    - pdf-parser.js applies flipY to all post positions (post.y is in flipped space where +y = north/up in UTM).
    - Bearing from post A to post B in PDF space (after flipY): bearingDeg = atan2(B.x - A.x, B.y - A.y) × (180/π). This produces 0° = north (+y direction = increasing northing after flipY).
    - Verify this convention by checking coordinate-calculator.js: the existing gpsBearing and cableSegmentBearingDeg functions in utm-calibrator.js and cable-builder.js use the same atan2(dx, dy) convention.

    The function must be browser-compatible: no fs, no Buffer, no Node-only globals.
