<interfaces>
From parser/geo/utm-calibrator.js:
```javascript
export function latLonToUtm(lat_deg, lon_deg); // → { easting, northing, zone }
export function utmToLatLon(easting, northing, zone); // → { lat, lon }
```

coordenadas postes siriu.txt format (semicolon-separated):
```
Poste 01; -27.97810488404128, -48.64053353230944;
Poste 85; -27.975700490555738, -48.631143602159746;
```
Parse: number = parseInt after "Poste ", lat = first float, lon = second float.

dxf-parser API (to be installed in this plan — for the fixture builder):
```javascript
import DxfParser from 'dxf-parser';
const dxf = new DxfParser().parseSync(dxfText); // → { header, tables, blocks, entities }
// entities is Array<entity> where entity.type, entity.layer, entity.position.x/y, entity.vertices[]
```

Poste INSERT filter: entity.type === 'INSERT' && entity.layer === 'Poste'
  → { x: entity.position.x, y: entity.position.y, block: entity.name }
TrechoSecundarioAereo LWPOLYLINE filter: entity.type === 'LWPOLYLINE' && entity.layer === 'TrechoSecundarioAereo'
  → { a: { x: vertices[0].x, y: vertices[0].y }, b: { x: vertices[last].x, y: vertices[last].y } }

CRITICAL — DO NOT scale coordinates by 1/1000. $INSUNITS=4 means "mm" in AutoCAD's table
but siriu.dxf entity coordinates are raw UTM meters (EXTMIN ~642260, EXTMAX ~742091).
DO NOT read $LATITUDE/$LONGITUDE from header — those are San Francisco defaults (37.795, -122.394),
not a georeference. Treat entity (x, y) as (easting, northing) in UTM zone 22S directly.
