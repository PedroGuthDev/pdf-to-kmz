      const header = new TextDecoder().decode(headerBytes);
      if (header.startsWith('AC1')) {

        statusEl.style.color = 'var(--error)';
        return;
      }

      try {
        statusEl.textContent = 'Processando DXF...';
        await regionLibrary.addRegion(name, file);
        statusEl.textContent = `Região "${name}" carregada com sucesso.`;
        statusEl.style.color = 'var(--success)';

    });

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file input → DXF parser | Untrusted file content; could be malformed or a binary DWG |
| IndexedDB → in-memory region | Retrieved region data feeds the pairing algorithm; malformed region could cause pairing failure but cannot execute arbitrary code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13-01 | Tampering | DXF file upload: user uploads a binary DWG instead of DXF | mitigate | Magic-byte check (AC1 prefix = binary DWG). User-visible error message in Portuguese with re-export instructions. Implemented in the upload handler per Pitfall #5 mitigation. |
| T-02-13-02 | Tampering | DXF file upload: user uploads a crafted DXF with extreme coordinate values | accept | dxf-parser library handles malformed input. pairPostsAgainstRegion fails strict-pair on implausible coordinates (tolerance check) and falls back to calculateCoordinates. No silent corruption. |
| T-02-13-03 | Denial of Service | User uploads a 100+ MB DXF file | accept | Browser file input size is not restricted. Personal tool, single user. dxf-parser parses synchronously; a very large file will block the UI thread briefly. For v1 this is acceptable — a loading indicator is shown during processing. |
| T-02-13-04 | Information Disclosure | calculateCoordinatesWithDwg warning objects exposed in result.warnings surfaced to DOM | accept | Warning text is shown to the user in the warnings list area (existing behavior for calc warnings). No PII in DWG warning objects — they contain post numbers and UTM distances. |
</threat_model>
