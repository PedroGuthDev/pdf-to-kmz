<objective>
Ship the N1+Viterbi-HMM accuracy iteration so the João Born sample meets G-2 (max < 10 m, 25+/33 < 5 m on 1-anchor) without regressing the Palhoça/Valmor G-1 baseline (max < 5 m, 11/11 < 5 m).

Two algorithmic improvements ship together because each one depends on the other being correct first:

1. **Viterbi-HMM symbol assignment (D-V-01/02/03 + D-SYM-01/02)** — Replace the beam-search body in `assignPolesGloballyByLabels()` with full Viterbi-HMM (~30 lines plain JS, no library). The transition term penalises |arcLen×scale − label_m|, which beam search width-8 lacks. Also tighten the cable-proximity candidate filter from 150 pt to 60 pt to cut João Born page 3 from 144 candidates to ~14 before Viterbi runs, with a per-post fallback warning when a legitimate post sits > 60 pt from the cable. After assignment lands, a post-hoc diagnostic emits a D-SYM-02 warning for any post whose final nearest-cable distance still exceeds 50 pt.

2. **N1 default-on for multi-sheet routes (D-N1-01..05, literal scope)** — Delete the `enableCableArcPlacer` opt-in boolean so the cable arc-length walk runs whenever the existing multi-sheet gate (`(overviewComposite || viewportBoxes.length >= 3) && cable && scale`) is satisfied. The multi-sheet gate itself is unchanged — Valmor (single-sheet, 2 viewport boxes) is NOT activated by this plan; that widening is a separate decision that requires its own gate evidence. Wire N1's anchor input to `post.x`/`post.y` (the now-Viterbi-correct pole symbol position) rather than `anchorX`/`anchorY`. Missing labels fall back through `augmentCrossPageDistances()`; tap poles are excluded; pages do not chain.

Before either ships, add a deterministic Valmor page-4 fixture to `post-positioning.test.mjs` (D-N2-01). The previous N2 attempt regressed Valmor page 4 because some Poste symbols were mis-assigned by greedy/beam matching, making `anchorX/anchorY` more accurate than `post.x/post.y` for those posts. The fixture proves where the mis-assignment happens and proves Viterbi fixes it before N1 starts trusting `post.x/post.y` as the arc anchor.

Purpose: Sequence (test fixture → Viterbi → N1 wiring) is non-negotiable. Each step depends on the previous being correct. G-1 (Valmor) is the gate after every step.
