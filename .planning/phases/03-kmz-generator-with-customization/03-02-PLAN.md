<action>
Create `parser/kml-builder.js` with `export function buildKml(posts, connections, options = {})`.

Internal helpers (not exported unless useful): `escapeXml(str)` replacing `&`, `<`, `>`, `"`, `'`.

Flow:
1. `const merged = mergeOptions(options)`; `const colors = resolveStyleColors(merged)`.
2. `const postByNum = new Map(posts.map(p => [p.number, p]))`.
3. Initialize `stats = { placemarkCount: 0, lineCount: 0, omittedNoGps: 0, skippedLines: 0 }` and `warnings = []`.
4. Emit XML prolog + `kml xmlns="http://www.opengis.net/kml/2.2"` + `Document` with `name` `pdf-to-kmz route`.
5. Shared styles `postPoint` and `routeLine`:
   - `postPoint`: IconStyle (color from `colors.iconColorKml`, scale `1`, href from `merged.iconHref`), LabelStyle (`colors.labelColorKml`, scale `merged.labelScale`).
   - `routeLine`: LineStyle (`colors.lineColorKml`, width `merged.lineWidth`).
6. For each post: if `lat == null || lon == null`, increment `omittedNoGps`, push `[kml-builder] post NN omitted (no GPS)`, skip placemark.
   - Else placemark: `name` = `Poste ${String(number).padStart(2,'0')}`; `description` = `Lat: {lat}, Lon: {lon}` (6+ decimal ok); `styleUrl` `#postPoint`; `Point` with `altitudeMode` `clampToGround`; `coordinates` = `{lon},{lat},0` (lon first).
7. For each `connections[]` entry: resolve `from`/`to` posts; if either missing GPS, increment `skippedLines`, warning `edge from→to skipped`, continue.
   - Line placemark: `name` = `Poste {from} → Poste {to}` (padded); `description` = `escapeXml(merged.lineDescription)`; `styleUrl` `#routeLine`; `LineString` with two coordinate tuples; same style for `gap` and `cross_page` (D-LN-02..04).
8. Update `stats.placemarkCount` / `stats.lineCount`; attach `warnings` to stats or return separately — return `{ kml, stats }` where `stats.warnings` is the array (D-API-02).

Do NOT chain posts by sorted number for lines.
</action>
<acceptance_criteria>
- `buildKml` is named export from `parser/kml-builder.js`
- KML contains `xmlns="http://www.opengis.net/kml/2.2"` and `Style id="postPoint"`
- Post with GPS produces `&lt;name&gt;Poste 07&lt;/name&gt;` for number 7
- `coordinates` in Point uses lon,lat order (lon before lat in string)
- One connection `{from:1,to:2}` with both GPS produces exactly one `LineString` placemark
- Post `{number:3,lat:null,lon:null}` increments `stats.omittedNoGps` and has no Point placemark
- `buildKml([], [], {})` returns valid empty Document (0 placemarks)
- `stats.warnings` is array (may be empty)
</acceptance_criteria>
</task>
