<task id="T-03-04-02" type="execute">
<title>Wire calc → lastCalcResult → download handler</title>
<read_first>
- index.html (calcBtn handler ~600–687, showWarnings, showStatus patterns)
- .planning/phases/03-kmz-generator-with-customization/03-UI-SPEC.md (Interaction States table)

</read_first>
<files>index.html</files>
<action>
Import in script module block: `buildKml`, `packageKmz`, `mergeOptions` from `./parser/pdf-parser.js` (or split imports if already importing calc from there).

Module-level: `let lastCalcResult = null; let lastKmzObjectUrl = null;`

After successful `calculateCoordinates` in calc handler:
- Set `lastCalcResult = { posts: calculatedPosts, connections, warnings: calcWarnings }`.
- Count posts with GPS: `placemarkEligible = posts.filter(p => p.lat != null && p.lon != null).length`.
- Enable `#downloadKmzBtn` if `placemarkEligible > 0`; else keep disabled.
- Hide or clear `#kmzStats` until download/build.

`downloadKmzBtn` click handler:
1. If no `lastCalcResult`, return.
2. Set button `disabled`, `aria-busy="true"`, text **Building KMZ…**.
3. Try:
   - `const opts = mergeOptions({});` (hardcoded defaults per D-API-03)
   - `const { kml, stats } = buildKml(lastCalcResult.posts, lastCalcResult.connections, opts);`
   - If `stats.placemarkCount === 0`: showStatus error **No posts with GPS — fix coordinates and recalculate.** — stop.
   - `const blob = await packageKmz(kml);`
   - Revoke prior `lastKmzObjectUrl` if set.
   - `const url = URL.createObjectURL(blob); lastKmzObjectUrl = url;`
   - Trigger download: `a.download = 'route.kmz'`, programmatic click.
   - Show `#kmzStats` with body: `{n} post placemarks · {m} route lines` (UI-SPEC template).
   - If `stats.omittedNoGps > 0`: style omitted line with warning colors; append to `#warningsList` li `[kmz] {n} posts omitted (no GPS coordinates).`
   - Append each `stats.warnings` to warnings list with `[kmz] ` prefix.
   - showStatus success: **KMZ ready — open in Google Earth.**
4. Catch: showStatus `Could not build KMZ: ${message}` or `Could not package KMZ file: ${message}` per UI-SPEC; do not clear `#outputPreview`.
5. Finally: restore button label **Download KMZ**, remove `aria-busy`, re-enable if eligible.

On new Calculate Route: revoke object URL; reset stats panel hidden.
</action>
<acceptance_criteria>
- After successful calc with ≥1 GPS post, `#downloadKmzBtn` is enabled
- Click produces browser download of `route.kmz`
- `#kmzStatsBody` shows placemark and line counts after successful build
- `[kmz]` prefix appears on KMZ warnings in `#warningsList`
- Loading state shows **Building KMZ…** and `aria-busy="true"`
- Build failure leaves output preview intact and sets error status
</acceptance_criteria>
</task>
