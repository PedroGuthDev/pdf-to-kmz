Layer name map (confirmed by skeleton run on real PDF):
- Numero_Poste  (OCG 51R) — red circles, fn=91 constructPath
- TEXTO         (OCG 52R) — sequential post numbers; CTM correlation required
- Distância_Poste (OCG 44R) — inter-post distances; â accent; normalize before compare
- Cabo Projetado  (OCG 43R) — cable route polyline; exact name with space

OPS constants (pdf.js 5.x, verified from src/shared/util.js):
- fn=10  save              — push graphics state
- fn=11  restore           — pop graphics state
- fn=12  transform         — concatenate CTM: args=[a,b,c,d,e,f]
- fn=49  showText          — text paint op (tracks CTM position)
- fn=50  showSpacedText    — text paint op (tracks CTM position)
- fn=70  beginMarkedContentProps — layer start WITH OCG ID: args=[tag, {id}]
- fn=71  endMarkedContent  — layer end
- fn=91  constructPath     — batched path: args=[opsArray, coordsArray]

PathOp types from parseConstructPath:
- { type: 'M', x, y }
- { type: 'L', x, y }
- { type: 'C', x1, y1, x2, y2, x3, y3 }
- { type: 'C2', x1, y1, x2, y2 }
- { type: 'C3', x1, y1, x2, y2 }
- { type: 'Z' }
- { type: 'R', x, y, w, h }

CRITICAL — CTM correlation approach (SKELETON.md "Text pipeline" row):
1. Walk page.getOperatorList() — track activeLayer via fn=70/71
2. Track CTM stack through fn=10 (save), fn=11 (restore), fn=12 (transform)
3. When fn=49 or fn=50 fires while activeLayer matches target — record CTM (e, f)
4. Walk page.getTextContent() items — each item.transform[4]=tx, item.transform[5]=ty
5. Match text items to recorded CTM positions (tx===e && ty===f within float tolerance)
6. Text items whose position matches belong to that layer

CRITICAL — Circle centroid approach (SKELETON.md A1 resolution):
- Circles are 4 cubic Bezier arcs; local center is (0,0)
- Circle page position = CTM (e, f) active at the fn=91 call in Numero_Poste layer
- Track CTM stack through fn=10/11/12 in the operator list walk
- When fn=91 fires while activeLayer === 'Numero_Poste': circle.x = ctm.e, circle.y = flipY(ctm.f, pageHeight)

CRITICAL — Diacritic normalization (SKELETON.md "Layer Name Normalization"):
```js
const normalizeName = s => s.normalize('NFD').replace(/[̀-ͯ]/g, '').toLowerCase();
```
Use this to match 'Distância_Poste' (â U+00E2) against expected names.

    The function signature is: `export async function extractLayerText(page, idToName)`

    STEP 1 — Walk the operator list to collect CTM positions per layer:
    - Maintain a CTM stack: initial state `{ a:1, b:0, c:0, d:1, e:0, f:0 }`.
    - fn=10 (save): push a copy of current CTM onto stack.
    - fn=11 (restore): pop the stack.
    - fn=12 (transform): concatenate args [a,b,c,d,e,f] into current CTM. For translation-only transforms (b===0, c===0), update e and f: `cur.e += args[4]; cur.f += args[5]`. For full matrix: apply full 2D affine concatenation.

    - fn=71 (endMarkedContent): set activeLayer = null.
    - fn=49 (showText) or fn=50 (showSpacedText): if activeLayer is not null, record `{ layer: activeLayer, e: ctm.e, f: ctm.f }` in a positions array.

    STEP 2 — Walk getTextContent() items:
    - Call `page.getTextContent()` (no includeMarkedContent needed — we do not use the markers here).
    - For each item where `item.str !== undefined`: tx = item.transform[4], ty = item.transform[5].
    - Find the matching recorded position where `Math.abs(pos.e - tx) < 0.5 && Math.abs(pos.f - ty) < 0.5`.
    - If a match is found, assign the item to that layer.

    Return value: `byLayer` object where keys are raw layer names (as stored in idToName) and values are arrays of `{ str: String, x: Number, y: Number }`. x = tx, y = ty (flipY is applied in pdf-parser.js, not here).

</threat_model>
