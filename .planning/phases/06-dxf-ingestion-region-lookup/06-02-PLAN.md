<objective>
Close the no-region failure path and the region-listing UI. When the first post's GPS falls inside no region's bounding box, the cascade caller (`coordinate-calculator-dwg.js`) must surface a structured `{ code:'NO_REGION', nearest:{ name, distanceKm } }` error (D-06) with the nearest region computed by haversine from the query GPS to each region's `bboxLatLon` centroid (D-07). Critically, the leaf `lookupByGps` and the hybrid wrapper MUST keep returning `null` on a miss so the cloud fallback is not short-circuited (A5 / Pitfall 5 — the load-bearing integration decision). Surface each region's GPS bbox in the upload dropdown (SC-5).

Purpose: Closes DXF-05 (fail loud with nearest-region hint, never silent wrong-region match) and DXF-07 (user can list ingested regions with GPS bboxes). The test also asserts DXF-04's POSITIVE lookup contract (a GPS inside a region's bbox returns that covering region) so the bbox index is proven to match on hits, not just to miss correctly. After this slice, a user who uploads a PDF whose GPS lands outside every ingested region gets a machine-readable NO_REGION result with a nearest-region distance hint instead of a confidently-wrong placement.
Output: NO_REGION synthesis at the cascade level, the hybrid wrapper left null-returning, a dropdown that shows GPS bboxes, and a test asserting the structured error, the positive in-bbox lookup, and the leaf-returns-null contract.
</objective>
